# 安全功能使用指南

## 1. 登入流程

### 標準登入（無 MFA）
1. 進入 https://localhost:5002/login
2. 填入帳號和密碼
3. 填入圖形驗證碼（点擊「重新整理驗證碼」獲得新的）
4. 點擊「登入」

### 啟用 MFA 的登入（雙因素驗證）

**首次設定 MFA**
1. 帳號管理頁面 → 點擊「建立我的 MFA 金鑰」
2. 使用 Google Authenticator 或 Microsoft Authenticator 掃描 QR Code
3. 輸入 App 顯示的 6 位數驗證碼
4. 確認啟用

**登入時**
1. 輸入帳號密碼 + 圖形驗證碼 → 登入
2. 系統進入「雙因素驗證」頁面
3. 打開 Authenticator App，輸入當前 6 位數
4. 點擊「確認」完成登入

## 2. 帳號管理（管理員操作）

### 建立新帳號

**位置**：帳號管理 → 建立新帳號區塊

**步驟**
1. 填入帳號（英數字、符號，1-64 字元）
2. 選擇顯示名稱（非必填）
3. 選擇角色：
   - **Admin**：系統管理員（完全權限）
   - **Operator**：操作員（版型、媒體、設備）
   - **Viewer**：檢閱者（唯讀權限）
4. 設定初始密碼（至少 8 字元，需含大/小寫、特殊符號）
5. 點擊「建立帳號」

### 停用/啟用帳號

**位置**：帳號清單 → 操作欄位

1. 找到目標帳號
2. 點擊「停用」或「啟用」按鈕
3. 已停用帳號使用者無法登入

### 重設密碼

**位置**：帳號清單 → 操作欄位

1. 點擊「重設密碼」
2. 輸入新密碼
3. 點擊「確認重設」
4. 帳號持有人下次登入須使用新密碼

### 更改角色

**位置**：帳號清單 → 角色欄位

1. 點擊帳號列的角色下拉菜單
2. 選擇新角色
3. 自動保存，無需確認

### 停用使用者的 MFA

**位置**：帳號清單 → 操作欄位

若 MFA 欄位顯示「已啟用」
1. 點擊「停用MFA」按鈕
2. 該使用者登入時不再需要 TOTP 驗證
3. 使用者需重新設定 MFA 時重複「首次設定 MFA」流程

## 3. 密碼政策

### 密碼要求
```
✓ 長度：最少 8 個字元
✓ 大寫字母：至少 1 個（A-Z）
✓ 小寫字母：至少 1 個（a-z）
✓ 特殊符號：至少 1 個（如 !@#$%^&*-_）

示例：Admin@123 ✅ | admin123 ❌ | Admin123 ❌
```

### 密碼有效期
- 密碼 180 天後過期
- 登入時提示須變更
- 前 3 次密碼禁止重複

### 密碼變更
1. 已登入狀態下，進入帳號管理其他頁面時若密碼已過期會提示
2. 獨立變更步驟（需已登入）

## 4. 帳號鎖定

### 什麼情況下帳號會被鎖定？
- **登入失敗 5 次**（傳入錯誤帳密）
- 帳號將鎖定 15 分鐘

### 如何解鎖？
- **自動解鎖**：等待 15 分鐘後再試
- **管理員解鎖**：重設密碼會同時清除鎖定

### 防暴力破解限制
- 登入端點每分鐘限制 10 次請求/IP
- 超過限制會返回 HTTP 429 Too Many Requests

---

## 5. 閒置自動登出

### 機制
- **閒置時間**：15 分鐘
- **偵測事件**：滑鼠移動、鍵盤輸入、點擊操作

### 使用者須知
- 15 分鐘無任何操作，系統自動登出
- 系統會顯示警示訊息
- 需重新登入方可繼續使用

## 6. 操作稽核（審計日誌）

### 位置
帳號管理 → 操作稽核 頁面

### 記錄內容

| 事件類型 | 記錄內容 |
|---------|--------|
| **登入** | 帳號、IP、時間、成功/失敗原因 |
| **MFA** | 設定、驗證、啟用/停用 |
| **帳號** | 建立、停用、重設密碼、角色變更 |
| **系統** | 初始化、完整性檢查 |

### 查詢日誌

1. 點擊「重新整理」取得最新日誌
2. 默認顯示最近 200 筆
3. 表格欄位說明：
   - **時間**：事件發生 UTC 時間（精確至秒）
   - **使用者**：執行操作的帳號
   - **操作**：執行的動作
   - **成功**：✅ 成功 / ❌ 失敗
   - **詳情**：具體說明或失敗原因
   - **IP**：發起操作的 IP 位址

### 驗證日誌完整性

1. 進入 帳號管理 頁面
2. 點擊「檢查日誌完整性」按鈕
3. 系統驗證所有日誌的雜湊鏈「1」
4. 顯示結果：
   - ✅ **完整**：日誌未遭篡改
   - ❌ **受損**：檢測到日誌被修改（重大安全事件）

## 7. 權限說明

### Admin（系統管理員）
```
可執行操作：
✓ 帳號管理（建立、停用、重設密碼、變更角色）
✓ MFA 管理（啟用/停用他人 MFA）
✓ 版型配置（建立、編輯、激活)
✓ 媒體管理（上傳、刪除）
✓ 設備管理（註冊、更新、設備)
✓ 操作稽核（檢視所有日誌、驗證完整性）

無法操作：
✗ 停用自己的帳號
✗ 停用唯一的管理員帳號（防誤刪）
```

### Operator（操作員）
```
可執行操作：
✓ 版型配置（建立、編輯、激活）
✓ 媒體管理（上傳、刪除）
✓ 設備管理（註冊、更新配置）
✓ 檢視設備狀態

無法操作：
✗ 帳號管理
✗ 操作稽核
```

### Viewer（檢閱者）
```
可執行操作：
✓ 檢視版型
✓ 檢視媒體清單
✓ 檢視設備狀態

無法操作：
✗ 建立/編輯版型
✗ 上傳/刪除媒體
✗ 其他修改操作
```

## 8. 一般常見問題

### Q: 忘記密碼怎麼辦？
A: 聯繫系統管理員，由管理員進行「重設密碼」操作。

### Q: 帳號被鎖定，無法登入？
A: 
- 如果是自己的帳號，聯繫管理員
- 管理員可透過「重設密碼」同時清除鎖定

### Q: 如何啟用 MFA？
A: 進入帳號管理頁面，點擊「建立我的 MFA 金鑰」並掃描 QR Code。

### Q: 遺失 Authenticator App 怎麼辦？
A: 由管理員進入帳號清單，點擊「停用MFA」，然後重新設定。

### Q: 日誌記錄包含什麼？
A: 時間戳記、使用者帳號、操作類型、目標、成功/失敗、IP 位址等。

### Q: 系統現在幾點（用於 TOTP 同步）？
A: 所有時間戳記使用 UTC (Coordinated Universal Time)，確保全球同步。

## 9. 故障排除

### 登入頁面顯示「圖形驗證已過期」
- **原因**：驗證碼超過 5 分鐘未使用
- **解決**：點擊「重新整理驗證碼」

### 登入失敗後顯示「帳號已鎖定」
- **原因**：帳號已因失敗登入 5 次而被鎖定
- **解決**：等待 15 分鐘後重試，或聯繫管理員

### TOTP 驗證碼顯示「不正確」
- **原因**：
  - 輸入的驗證碼已過期（30 秒內有效）
  - 手機與伺服器時間不同步
- **解決**：
  1. 同步手機時鐘
  2. 使用最新的 6 位驗證碼
  3. 若問題持續，管理員停用 MFA 重新設定

### 15 分鐘內自動登出
- **原因**：系統無任何使用者交互 15 分鐘以上
- **解決**：重新登入即可恢復

### 無法檢視帳號管理頁面
- **原因**：使用者角色非 Admin
- **解決**：該功能限管理員使用

## 10. 安全最佳實踐

### ✅ 應該做
- 使用複雜密碼（混合大小寫、數字、符號）
- 定期更改密碼
- 啟用雙因素驗證 (MFA)
- 保管好 Authenticator App
- 記住「重要設備」的信息以備緊急恢復
- 定期檢查操作稽核日誌

### ❌ 不應該做
- 寫下密碼或分享給他人
- 在公用電腦上勾選「記住我」
- 使用容易猜測的密碼
- 在不安全的網路上登入
- 洩漏 Authenticator App 二維碼
- 直接傳輸密碼給管理員
