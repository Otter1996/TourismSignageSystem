# 觀光局電子看板系統 操作手冊

## 1. 文件目的
本手冊提供管理人員與維運人員在日常作業中使用「觀光局電子看板管理系統」的完整操作步驟，包含：
- 系統啟動
- 遊客中心管理
- 版型配置與推送
- 媒體素材管理
- 設備管理
- 播放端檢查
- 常見問題排除

## 2. 系統角色與網址
- 管理後台（Signage.App）：用於配置版型、管理素材、管理設備
- 後端 API（Signage.Server）：提供資料與命令介面
- 播放端頁面（Display）：模擬看板播放畫面
- 登入頁面（Login）：帳號密碼 + 圖形驗證

開發環境常用網址：
- 本機 localhost（https 啟動設定）
   - 後端 API：https://localhost:7245
   - 管理後台：https://localhost:7199
   - 播放端：https://localhost:7199/display
   - 登入頁：https://localhost:7199/login
- Codespaces / Dev Container（建議使用轉送後網址）
   - 後端 API：https://scaling-carnival-q4jwvvq94p7f9wv-5269.app.github.dev
   - 管理後台：https://scaling-carnival-q4jwvvq94p7f9wv-5179.app.github.dev
   - 播放端：https://scaling-carnival-q4jwvvq94p7f9wv-5179.app.github.dev/display
   - 登入頁：https://scaling-carnival-q4jwvvq94p7f9wv-5179.app.github.dev/login

## 2.1 帳號與權限設計
- 角色分級：
   - Admin：可建立帳號、查看稽核、執行全部管理操作
   - Operator：可執行版型/媒體/設備/推送等營運操作
   - Viewer：僅可查閱資料
- 帳號建立：僅 Admin 可於後台「帳號管理」頁建立新帳號
- 預設管理員：`admin / Admin@123`（首次上線建議立即更改）

## 2.2 密碼政策
- 密碼複雜度：至少 8 碼，且需包含
   - 至少一個大寫字母
   - 至少一個小寫字母
   - 至少一個特殊符號
- 密碼到期：180 天
- 歷史密碼限制：新密碼不可與前三組相同

## 3. 環境需求
- .NET 10 SDK
- 可使用終端機（Linux/macOS/Windows 皆可）
- 建議使用最新版 Chrome 或 Edge
- SQLite 資料庫檔案會由系統自動建立，預設位置為 `Signage.Server/Data/signage.db`

## 4. 系統啟動流程
在專案根目錄下，開兩個終端機視窗：

### 4.1 啟動後端 API
```bash
cd Signage.Server
dotnet restore
dotnet run --launch-profile https
```

若在 Codespaces / Dev Container 透過 Port Forward 使用，建議改用：
```bash
dotnet run --launch-profile http
```

若啟動成功，應看到類似訊息：
- Now listening on: https://localhost:7245

### 4.2 啟動管理後台
```bash
cd Signage.App
dotnet restore
dotnet run --launch-profile https
```

若在 Codespaces / Dev Container 透過 Port Forward 使用，建議改用：
```bash
dotnet run --launch-profile http
```

若啟動成功，應看到類似訊息：
- Now listening on: https://localhost:7199

### 4.3 開啟頁面
- 管理後台首頁： https://localhost:7199
- 播放端頁面： https://localhost:7199/display

## 5. 管理後台操作

### 5.0 登入流程（含圖形驗證）
1. 前往登入頁。
2. 輸入帳號與密碼。
3. 輸入圖形驗證碼。
4. 點擊登入。
5. 若密碼逾 180 天，系統會提示需更新密碼。

### 5.1 進入前準備
1. 開啟管理後台首頁。
2. 在左側「遊客中心清單」點選一個中心。
3. 若未選擇中心，主畫面會顯示提示，無法進行完整配置。

### 5.2 遊客中心管理
位置：左側欄位下半部

新增中心：
1. 在「新增遊客中心名稱」輸入名稱。
2. 點擊「確認新增」。
3. 成功後會出現在「遊客中心清單」。

編輯中心：
1. 在目標中心右側點擊編輯圖示。
2. 修改名稱後點擊「確認修改」。
3. 若名稱重複，系統會阻擋並提示。

刪除中心：
1. 在目標中心右側點擊刪除圖示。
2. 於確認視窗點擊「確認刪除」。
3. 系統會刪除該中心（示範版中為前端記憶體資料）。

### 5.3 版型配置（版型配置頁）
位置：左側選單「管理功能 > 版型配置」

操作步驟：
1. 先選擇遊客中心。
2. 在「選擇或新建版型」中選取既有版型，或點「新建版型」。
3. 設定「看板版型」：
   - 三段式 (30/40/30)
   - L型資訊模式
   - 全螢幕模式
4. 填入內容欄位：
   - 頂部：跑馬燈文字
   - 中部：影片 URL
   - 底部：圖片 URL
   - 若為 L 型，另有左側內容與右側內容
5. 點擊「儲存設定」。
6. 若要推送到設備，點擊「推送至看板」。

畫面右側為 16:9 終端模擬區，可即時預覽版型效果。

### 5.4 媒體庫管理（媒體庫頁）
位置：左側選單「管理功能 > 媒體庫」

新增素材：
1. 選擇媒體類型（圖片、影片、文字）。
2. 輸入媒體名稱。
3. 輸入可公開存取的媒體 URL。
4. 圖片/影片可設定播放時長。
5. 點擊「新增素材」。

素材操作：
1. 右側可透過「全部 / 圖片 / 影片」篩選。
2. 在列表中可：
   - 點複製圖示：複製 URL
   - 點刪除圖示：移除素材
3. 新增完成後可在下方預覽區確認素材內容。

### 5.5 設備管理（設備管理頁）
位置：左側選單「管理功能 > 設備管理」

註冊設備：
1. 輸入設備名稱（必填）。
2. 輸入設備 IP（選填）。
3. 點擊「註冊設備」。

設備監控：
1. 查看在線/離線/總數摘要。
2. 可用「全部 / 在線 / 離線」篩選。
3. 在設備卡片或表格執行：
   - 推送配置
   - 刷新狀態
   - 移除設備

### 5.6 帳號管理（Admin）
位置：左側選單「管理功能 > 帳號管理」

建立帳號：
1. 輸入帳號。
2. 輸入顯示名稱。
3. 選擇角色（Admin / Operator / Viewer）。
4. 輸入符合複雜度規則的初始密碼。
5. 點擊「建立帳號」。

### 5.7 稽核紀錄（Admin）
位置：左側選單「管理功能 > 操作稽核」

可查看：
1. 登入成功/失敗紀錄（含 IP、結果、原因）。
2. 帳號建立、密碼更新。
3. 版型、媒體、設備、推送等關鍵操作。

## 6. 播放端操作
網址： https://localhost:7199/display

操作方式：
1. 開啟播放端頁面。
2. 在上方「選擇版型」切換要播放的版型。
3. 確認播放畫面符合預期：
   - 三段式：跑馬燈 + 影片 + 圖片
   - L 型：右側影片 + 左側資訊
   - 全螢幕：背景影片 + 疊加跑馬燈
4. 觀察底部狀態列（版型名稱、更新時間、連線狀態）。

## 7. 日常作業建議流程
1. 選擇遊客中心。
2. 先在媒體庫新增或確認素材 URL。
3. 進入版型配置，更新版型內容。
4. 在右側模擬區核對畫面。
5. 執行推送至設備。
6. 到設備管理確認設備在線狀態。
7. 最後在播放端檢查實際顯示。

## 8. 常見問題與排除

### Q1：後台可以開啟，但看不到資料
- 先確認已選擇左側遊客中心。
- 後端目前已使用 SQLite 持久化資料；若資料未出現，請確認後端 API 是否正常啟動，且 `Signage.Server/Data/signage.db` 可寫入。

### Q2：影片或圖片不顯示
- 先在瀏覽器直接打開素材 URL，確認是否可公開存取。
- 確認影片格式建議為 MP4/WebM。
- 若來源站台有防盜鏈或跨網域限制，可能無法直接播放。

### Q3：設備一直離線
- 確認設備端是否有定期送出心跳。
- 確認網路與 API 可連通。
- 示範模式中狀態可能由前端模擬，重整後可能重置。

### Q4：按下推送沒有實際生效
- 先確認後端 API 已啟動。
- 再確認前端設定中的 API BaseUrl 是否指向正確位址。
- 若頁面是示範模式，按鈕可能只做本地流程演示。

### Q5：為什麼無法登入？
- 先確認圖形驗證碼正確且未過期。
- 檢查帳號是否停用或密碼是否錯誤。
- 請由 Admin 至「操作稽核」確認失敗原因與時間。

## 9. 維運注意事項
- 目前系統已使用 SQLite 儲存核心資料（遊客中心、版型、媒體、設備、推送命令、帳號、稽核紀錄）。
- 正式上線若需多人並發、備援或集中式管理，可再升級為 SQL Server / PostgreSQL。
- 建議加入帳號權限控管與操作稽核紀錄。
- 建議加入監控與告警（API 健康檢查、設備心跳逾時告警）。
- 建議為素材來源建立白名單，避免不可信 URL。
